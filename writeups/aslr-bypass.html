Research // RE-002

ASLR Bypass

Executive Summary

"Operasi ini mendemonstrasikan pengambilalihan kontrol sistem melalui eksploitasi memori pada biner sistem SUID. Tantangan utama adalah melewati proteksi ASLR pada Linux 32-bit menggunakan teknik ret2libc, membuktikan bahwa proteksi memori dapat dilampaui melalui analisis entropi."

01

/ Initial Access: The Unintended Leak

Penemuan awal bermula dari analisis file JavaScript (app.js). Ditemukan kredensial SSH yang tertanam secara tidak sengaja dalam kode aplikasi, memberikan akses shell awal.

# Extracting secrets from client-side JS
$ grep -iE "user|pass|secret" app.js
const SSH_USER = "svc_account";
const SSH_PASS = "S3cur3_P@ssw0rd_2026";
$ ssh svc_account@target.internal
svc_account@server:~$ id
uid=1000(svc_account) gid=1000(svc_account) groups=1000...
02

/ Vulnerability Research: Buffer Overflow

Target utama eskalasi adalah biner SUID secure_log. Analisis checksec menunjukkan NX aktif, mencegah eksekusi shellcode di stack.

Checksec Output:
Arch:i386-32-bit
Canary:Disabled
NX:Enabled
PIE:Disabled

Stack Visualizer

Buffer (260b)

03

/ Ret2Libc & ASLR Bypass Execution

Struktur payload disusun untuk memanggil system() dengan argumen /bin/sh. Brute-force dilakukan untuk menebak alamat libc pada sistem 32-bit.

[*] Brute forcing libc base address...
Attempt 100: Segfault
Attempt 200: Segfault
Attempt 260: [!] HIT! Library base found.
svc_account@server:~$ ./exploit
root# whoami && id
root (uid=0)

04 LESSONS LEARNED

Secure API

Ganti strcpy() dengan strncpy() untuk mencegah memory corruption.

Modern Compiler

Aktifkan Canary dan PIE untuk menambah lapisan perlindungan biner.

Binary Hardening

Gunakan 64-bit untuk meningkatkan entropi ASLR dan mencegah brute force.
